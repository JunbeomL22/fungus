# Product Context

## Purpose
The Fungus project is for developing java application for low-latent trading system

## Problems it Solves
1. Demonstrates Maven build and dependency management for Java projects

## How it Should Work
1. Benchmarks should be executable and produce measurable results
2. The project should be easily buildable using Maven (via mvn-compile.bat)
3. Dependencies should be properly managed through pom.xml


## User Experience Goals
1. Easy to understand and modify benchmark implementations
2. Simple build and run process
3. Clear output of benchmark results
4. Well-documented code and configuration
